# Drawcast 기본 Rubric (VLM 채점)

채점자: Codex CLI(GPT-5.4, vision). 입력으로 **원 질문과 렌더된 PNG만** 받는다. scene JSON은 의도적으로 주어지지 않는다 — 이 rubric은 사람이 화면에서 보는 것과 동일한 신호로만 품질을 판정한다(구조·레이블의 기계적 확인은 별도의 JSON 룰 베이스 트랙 `runner/metrics.ts`가 담당하므로 여기서 JSON을 참조하면 두 트랙의 신호가 섞여 오염된다).
응답은 반드시 아래 **JSON 포맷**으로만 출력한다. 자연어 설명은 `notes` 필드에 담는다.

## 5개 채점 축

### 1. structure (0~5)
질문이 요구하는 주요 엔티티(노드)와 관계(엣지)가 모두 존재하는지.

| 점수 | 기준 |
|---|---|
| 5 | 모든 필수 요소 + 유의미한 추가 요소 (e.g. 에러 분기, 경계 케이스) |
| 4 | 모든 필수 요소가 정확히 존재 |
| 3 | 1~2개 필수 요소 누락 또는 불필요한 추가로 어수선 |
| 2 | 핵심 구조 일부 누락 |
| 1 | 구조 이해가 미흡, 다수 누락 |
| 0 | 질문이 요구한 구조와 완전히 다름 |

### 2. labels (0~5)
각 노드·엣지 레이블이 **의미적으로 정확하고 구체적**인지. (한국어·영어 혼용 OK)

| 점수 | 기준 |
|---|---|
| 5 | 모든 레이블이 정확·간결·구체적. 오타 없음 |
| 4 | 대부분 정확. 사소한 모호함 1~2곳 |
| 3 | 일부 레이블이 부정확·빈약·지나치게 일반적 |
| 2 | 다수 레이블이 부정확하거나 비어 있음 |
| 1 | 대부분 레이블 없음/틀림 |
| 0 | 레이블 자체가 없거나 의미가 통하지 않음 |

### 3. layout (0~3)
겹침 없이, 일관된 흐름 방향(위→아래 / 좌→우 등)으로, 시각적으로 따라가기 쉬운지.

| 점수 | 기준 |
|---|---|
| 3 | 겹침 없음, 흐름 방향 일관, 정렬 깔끔 |
| 2 | 대체로 괜찮음, 정렬 일부 어긋남 |
| 1 | 읽기 어려움, 겹침 있음, 방향 섞임 |
| 0 | 공간 구조가 엉켜 해석 불가 |

### 4. readability (0~3)
글자 크기·색 대비·시각적 클러터 관점에서 한눈에 읽히는지.

| 점수 | 기준 |
|---|---|
| 3 | 글자·대비·여백 모두 무리 없이 읽힘 |
| 2 | 대체로 읽힘, 사소한 이슈 |
| 1 | 일부 글자가 잘림/겹침/작음 |
| 0 | 상당 부분 읽을 수 없음 |

### 5. intent_fit (0~5)
원 질문의 **의도**와 결과가 매치되는지. 구조 점수가 높아도 질문이 요구한 관점(예: "시퀀스"인데 박스만 가득)과 다르면 낮게.

| 점수 | 기준 |
|---|---|
| 5 | 질문 의도를 정확히 이해 + 의미 있는 디테일 |
| 4 | 의도 반영, 사소한 누락 |
| 3 | 의도는 맞지만 핵심 디테일 누락 |
| 2 | 부분적으로만 일치 |
| 1 | 표면적으로만 관련 |
| 0 | 질문과 무관 |

## 응답 포맷 (필수)

```json
{
  "structure": 4,
  "labels": 5,
  "layout": 3,
  "readability": 2,
  "intent_fit": 4,
  "major_issues": ["결제 실패 분기 누락"],
  "minor_issues": ["노드 3-4가 약간 겹침"],
  "notes": "전체적으로 의도는 잘 반영했으나 결제 실패 처리가 빠져 실무 품질에는 못 미침."
}
```

추가 규칙:
- 점수는 정수만 사용 (소수점 금지)
- `major_issues`는 1점 이상 감점 사유, `minor_issues`는 감점 없는 지적
- `notes`는 3줄 이내 한국어
- JSON 외의 텍스트를 출력하면 채점 실패로 간주

## 가중치와 정규화

최종 점수:

```
weighted = Σ(weight_i × score_i)
max      = Σ(weight_i × max_score_i)   # max_score: structure=5, labels=5, layout=3, readability=3, intent_fit=5
total    = weighted / max              # 0.0 ~ 1.0
```

질문의 `rubric_weights`가 축을 생략하면 **1.0**로 간주한다.

## 판정 기준 (집계용)

- **pass**: `total >= 0.70` **그리고** 모든 축이 **2점 이상**
- **fail**: `total < 0.50` **또는** 어느 축이든 **1점 이하** (0 또는 1)
- **borderline**: 그 외

축 하나라도 1점(3점 스케일은 33%, 5점 스케일은 20%) 이하로 떨어지면 "읽기 어려움/구조 누락" 같은 배송 불가 수준이라 보고 fail로 치환한다. 총점이 아무리 높아도 layout=1 또는 readability=1인 결과는 제품 품질 기준으로 통과 아님.

집계 리포트에서 `pass_rate = pass_count / total_runs`로 표시한다.
